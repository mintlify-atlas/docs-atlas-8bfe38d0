---
title: Quickstart
description: Get a working violence detection prediction in under five minutes.
---

<Steps>
  <Step title="Check prerequisites">
    Before you begin, make sure you have the following ready:

    - **Python 3.8 or later** — verify with `python --version`
    - **pip** — included with Python 3.8+
    - **Model weights** — two weight files must be present before starting the server:

    | File | Location |
    |------|----------|
    | `tsn_masked_best.pth` | `model/tsn_masked_best.pth` (relative to repo root) |
    | `yolov8n.pt` | repo root (`./yolov8n.pt`) |

    <Warning>
      The server will fail to start if either weight file is missing. `tsn_masked_best.pth` is the trained TSN classifier; `yolov8n.pt` is downloaded automatically by Ultralytics on first run if it is not present, but placing it in the root directory avoids the download delay.
    </Warning>
  </Step>

  <Step title="Clone the repo and install dependencies">
    <CodeGroup>
      ```bash Clone
      git clone https://github.com/ashlin311/violence-detection-system.git
      cd violence-detection-system
      ```

      ```bash Create virtual environment
      python -m venv .venv
      source .venv/bin/activate   # Windows: .venv\Scripts\activate
      ```

      ```bash Install dependencies
      pip install -r requirements.txt
      ```
    </CodeGroup>

    The `requirements.txt` installs the following packages:

    ```text requirements.txt
    flask
    flask-cors
    torch
    torchvision
    ultralytics
    opencv-python
    numpy
    Pillow
    ```

    <Tip>
      On machines without a CUDA-capable GPU, PyTorch will run inference on the CPU automatically — no extra configuration is needed.
    </Tip>
  </Step>

  <Step title="Start the server">
    ```bash
    python app.py
    ```

    The Flask server binds to `0.0.0.0` and listens on **port 5001** by default.

    ```
     * Running on http://0.0.0.0:5001
    ```

    To use a different port, set the `PORT` environment variable before starting:

    ```bash
    PORT=8080 python app.py
    ```

    <Note>
      Port 5001 was chosen deliberately to avoid conflicts with local tooling that commonly occupies port 5000.
    </Note>

    Confirm the server is healthy:

    ```bash
    curl http://localhost:5001/health
    ```

    ```json
    {"status": "ok"}
    ```
  </Step>

  <Step title="Make a prediction">
    Send a video file to `POST /predict` using the `video` form field. The endpoint accepts the following formats: `.mp4`, `.avi`, `.mov`, `.mkv`, `.webm`, `.m4v`.

    ```bash
    curl -X POST http://localhost:5001/predict \
      -F "video=@/path/to/your/clip.mp4"
    ```

    The server samples **8 equally-spaced segments** from the video (deterministic middle-frame sampling per segment), applies a YOLOv8n person mask to each frame, and runs the TSN classifier.

    A successful response looks like this:

    ```json
    {
      "prediction": "Violent",
      "confidence": 0.9173,
      "person_count": 2,
      "most_active_frame": 4,
      "processing_time": 1.84,
      "original_frame": "<base64-encoded JPEG>",
      "masked_frame": "<base64-encoded JPEG>",
      "gradcam_frame": "<base64-encoded JPEG>"
    }
    ```

    | Field | Type | Description |
    |-------|------|-------------|
    | `prediction` | `string` | `"Violent"` or `"Non Violent"` |
    | `confidence` | `number` | Softmax probability for the predicted class (0 – 1) |
    | `person_count` | `integer` | Maximum number of persons detected in any single sampled frame |
    | `most_active_frame` | `integer` | Index (0-based, out of 8) of the segment with the highest fight-class score |
    | `processing_time` | `number` | Wall-clock seconds taken for inference |
    | `original_frame` | `string` | Base64-encoded JPEG of the most active frame, unmodified |
    | `masked_frame` | `string` | Base64-encoded JPEG with background dimmed (30 % brightness) outside detected persons |
    | `gradcam_frame` | `string` | Base64-encoded JPEG of the Grad-CAM heatmap overlaid on the masked frame (`null` on error) |

    <Note>
      All three frame fields (`original_frame`, `masked_frame`, `gradcam_frame`) are standard base64-encoded JPEGs. Prefix them with `data:image/jpeg;base64,` to embed them directly in an HTML `<img>` tag.
    </Note>

    #### Error responses

    | Status | Condition |
    |--------|-----------|
    | `400` | No `video` field in the request, empty filename, unreadable video, or codec error |
    | `500` | Unexpected server-side error (details in `error` and `error_type` fields) |
  </Step>

  <Step title="Open the web client">
    The repository ships a browser-based client in the `client/` directory. Open `client/index.html` directly in any modern browser — no build step required.

    ```bash
    open client/index.html        # macOS
    xdg-open client/index.html    # Linux
    start client/index.html       # Windows
    ```

    The client automatically targets `http://<hostname>:5001`. When opened from the local filesystem (`file://`) it falls back to `http://127.0.0.1:5001`.

    <CardGroup cols={2}>
      <Card title="Upload Video" icon="upload">
        Click **Upload Video** to pick a file from disk. Supported formats: `.mp4`, `.avi`, `.mov`, `.mkv`, `.webm`, `.m4v`. The analysis overlay displays the prediction, confidence score, person count, and all three frame visualizations.
      </Card>
      <Card title="Live Camera Scan" icon="video">
        Click **Start Camera** to grant webcam access. The client records an **8-second clip** (after a 3-second countdown) and sends it to the same `/predict` endpoint automatically.
      </Card>
    </CardGroup>
  </Step>
</Steps>
